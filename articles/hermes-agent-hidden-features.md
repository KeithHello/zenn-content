# チャットだけ？Hermes Agentの隠れた真価 — 52K StarのAIが本当にできること

## はじめに

Hermes Agent。GitHub 5.2万 Star。NousResearch が開発。名前くらい聞いたことある人も多いだろう。

インストールした人の大半は、これを「ターミナルで動く ChatGPT クローン」として使っている。だがそれは、スマホで電話しかかけていないのと同じだ。本稿では、[hermesai 中文社区](https://hermesai.top)が公開した10の隠れ機能の中から、実際の業務効率を変えた7つに絞って紹介する。

## 対象読者

- Hermes Agent をインストールしたが深く使えていない人
- AI エージェントの実用的な活用方法を探している人
- ChatGPT/Claude の「セッションが切れる問題」に悩んでいる人

## この記事で得られること

- Hermes を「チャットボット」から「個人AI OS」にアップグレードする方法
- 7つの機能の具体的な使い方とユースケース
- Hermes の得意分野と苦手分野の明確な線引き

## 結論（先出し）

Hermes Agent は Q&A ツールではない。**長期記憶・タスク実行・マルチプラットフォーム常駐・MCP拡張**の4本柱で構成される個人AIオペレーティングシステム。チャットしか使っていないなら、その8割を浪費している。

---

## 本文

### 1. 長期記憶 — 「毎回同じ説明」から解放される

多くの AI ツールはステートレスだ。会話が終われば記憶も消える。Claude はセッションをまたげない。ChatGPT の Memory 機能は限定的で手動管理が必要。

Hermes の長期記憶は自動かつ持続的だ。「短く答えて」と一度言えば、3ヶ月後も覚えている。プロジェクト名、好みの文体、よく使うツール。すべて蓄積され、越しに賢くなる。

```
あなた: プロジェクト名は「ナイトフォール」で統一して
Hermes: 了解。ナイトフォール、記録しました
```

これがクロスプラットフォームで有効なのも見逃せない。WeChat で伝えた設定が Telegram でも生きる。

### 2. セッション復元 — ターミナルが落ちても安心

```
hermes --continue
```

たったこれだけで前回の会話が完全復元。セッション名をつければ後から検索もできる。

### 3. 実行者としての Hermes

多くのユーザーが誤解している最大のポイント。Hermes は「やり方を教える」AI ではなく、「やってくれる」AI だ。

デスクトップの大容量ファイルを検索する？コマンドを提案するのではなく、実際に実行して結果を返す。テキストファイルを結合する？同じく。やるべきことを指示すれば、あとは全て自動。

### 4. 全プラットフォーム常駐

WeChat、Feishu、Telegram、Discord、Slack、DingTalk に同時接続。AI アシスタントがすべてのメッセージアプリに常駐している状態を作れる。

設定は `hermes gateway setup` をプラットフォームごとに1回実行するだけ。以降は統一的にあなたの情報を保持する。

### 5. 自然言語スケジューリング

「毎朝9時に昨日のAIニュースを3つにまとめて送って」がそのまま cron ジョブになる。YAML も cron 式も不要。

### 6. MCP で無限拡張

Model Context Protocol（MCP）は Hermes をプラットフォームに変えるエコシステムだ。Notion、GitHub、Google カレンダーなど様々なツールと接続でき、機能は無限に拡張可能。

### 7. マルチエージェント並列

複雑な調査タスクを自動で分割し、複数のサブエージェントが並列処理。逐次処理と比べて大幅な時短になる。

---

## まとめ

Hermes Agent は **「知っている・動ける・どこにでもいる」** という3つの特性を同時に満たす稀有なツールだ。

ただし万能ではない。コーディング補助なら Cursor や Claude Code、純粋な Q&A なら ChatGPT の方が向いている。Hermes の真価は「ツールとツールの間をつなぎ、コンテキストを維持し続ける」点にある。

チャットだけで満足しているなら、今すぐ `hermes --help` を打ってみることをおすすめする。そこには別世界が広がっている。
