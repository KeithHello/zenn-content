## はじめに

Hermes Agentをインストールして動かし始めたものの、「昨日教えたことを今日は覚えていない」「サブエージェントにタスクを任せたら壊滅的」「設定ファイルが多すぎて混乱」──そんな経験はありませんか。

実はこの3つの問題、すべて**Hermesの設計思想を正しく理解すれば一発で解決**します。本記事では、実際に踏み抜いた地雷と、そこから得た実践的ノウハウを凝縮してお届けします。

## 対象読者

- Hermes Agentを導入したばかりのエンジニア
- 記憶機能をうまく使えていない方
- マルチエージェントの挙動に困惑している方
- 「とにかく最短で使いこなしたい」方

## この記事を読むメリット

- エージェントに「確実に」記憶させる方法がわかる
- スキルを効果的に作成・管理する運用フローが身につく
- サブエージェントを使いこなすための必須ルールが理解できる
- 本番環境でのトラブルを未然に防げる

## 結論（先出し）

Hermes Agentの本質は**3つ**：

1. **記憶**：「忘れた」問題を解決──ただし大事なことは**明示的に**指示する
2. **スキル**：「これどうやるんだっけ」を解決──繰り返しパターンを**SOP化**する
3. **サブエージェント**：「1人じゃ無理」を解決──ただし**完全なコンテキスト**が必須

この3つを押さえれば、初心者が直面する問題の8割は消えます。

## 本文

### 第1章：記憶システムの真実

**最大の誤解**：「エージェントは会話内容を全部覚えている」

→ **違います。** Hermesは**エージェント策定型記憶**を採用。エージェント自身が「重要」と判断した情報だけを長期記憶に保存します。これはバグではなく、トークン節約とノイズ防止のための意図的な設計です。

**正しい覚えさせ方：**

```
私の好みを記憶して：すべてのコードはPython 3.11を使う。3.12や3.13は使わない。
```

```
現在の進捗を保存して：データクレンジングと特徴量エンジニアリングが完了。次はモデル学習。
```

**記憶ファイルの使い分けが最重要：**

| ファイル | 役割 | 記述者 |
|----------|------|--------|
| SOUL.md | 永続的なルール・行動指針 | あなた |
| USER.md | 好み・習慣 | エージェント |
| MEMORY.md | 作業メモ・環境情報 | エージェント |

> 🔴 SOUL.mdに書くべき固定ルールをMEMORY.mdに絶対に入れないこと。MEMORY.mdはエージェントが自動的に整理・圧縮する可能性があります。

長期間の作業では、`TASK_STATUS.md` をプロジェクトディレクトリに作成してもらうのが最も確実な状態保持方法です。

### 第2章：スキルシステム — 経験をSOPに昇華

スキルとはコードでもプラグインでもなく、「**再利用可能な標準作業手順書**」です。

優れたスキルの4要素：

1. **トリガー条件**：いつ使うべきか
2. **操作手順**：具体的に何をするか
3. **検証方法**：成功したかどうかの判断基準
4. **注意事項**：過去に踏んだ罠

**初心者が使うべき4つのコマンド：**

新規作成：「このデータ処理フローをdata-pipelineというスキルとして保存。トリガー条件・手順・検証方法・注意点を含めて。」

記憶から昇華：「デプロイ手順をdeploy-workflowというスキルにしてください。」

ピンポイント修正：「stock-daily-reportスキルの注意点に『金曜の引け後データは土曜朝まで遅延する可能性あり』を追加。」

定期監査：「全スキルをレビュー。重複は統合、不明瞭なものは書き直し、古いものは更新。先にレポートを出して。」

### 第3章：マルチエージェント — 並列処理の真実

**並列数の黄金律：2つから始め、最大3つまで。**

理論上の最大並列数を追うより、コスト管理とコンテキスト品質がはるかに重要です。

**初心者が最も踏む地雷：サブエージェントは会話履歴を継承しない。**

サブエージェントは真っ新なコンテキストから起動します。必要な情報はすべて `context` フィールドに明示的に渡す必要があります：

```python
delegate_task(
  goal="api/handlers.py の TypeError を修正",
  context="""
  ファイルパス: /home/user/myproject/api/handlers.py
  エラー: 47行目 TypeError: 'NoneType' object has no attribute 'get'
  原因: parse_body() が Content-Type ヘッダー不在時に None を返す
  プロジェクト: Python 3.11 + Flask
  """
)
```

AGENTS.md/SOUL.md に役割分担を明記するのが最も効果的です。Planner / Executor / Reviewer の3役に分けることを推奨します。

### 第4章：本番環境チェックリスト

- `hermes doctor` でエラーなしを確認
- APIキーは `.env` に記述（`config.yaml` には絶対に入れない）
- `GATEWAY_HEARTBEAT=true` を設定
- ユーザーホワイトリストを設定（`TELEGRAM_ALLOWED_USERS`）
- 承認モードを設定（`smart` または `manual`）
- サーバータイムゾーンを `timedatectl` で確認
- Cronジョブは本番投入前に必ず手動検証

## まとめ

初心者が直面する問題の8割は「記憶は自動だと思い込む」と「サブエージェントにコンテキストを渡さない」に起因します。まずは **「長期記憶に保存して」** と **「contextに全部書く」**、この2つから始めてください。

それだけで、Hermesとの付き合い方が劇的に変わります。
